⑤ Notes

How the tools relate

Brief observations on how each tool sits relative to Canary Lab and to each other. The author's read of public docs as of May 2026; not vendor statements.

Playwright Healer
Different layer
Healer patches test code — selectors, assertions, waits — using DOM, console, and network signals from a replay. Canary Lab doesn't patch tests; it captures system context so an external coding agent can work on the implementation.
Currents.dev
Different audience
Cloud-side history and flakiness analytics for teams running Playwright in CI. Canary Lab focuses on the single-developer local loop. Both can coexist in the same project.
Octomind
Different timing
Octomind generates and maintains tests at authoring time. Canary Lab adds run-time context at failure time. The two cover different phases of the test lifecycle.
Mabl / QA Wolf
Different operating model
Vendor-managed cloud SaaS where the vendor (or the vendor's AI) writes and maintains tests. Canary Lab is OSS, local, and assumes you write the tests.
Tilt
Adjacent space
Tilt orchestrates Kubernetes services for the dev loop with hot reload. Canary Lab orchestrates plain processes for the test loop with per-run lifecycle. Same verb, different runtime assumptions.
